# 滚齿工艺参数说明文档

本文档用于说明：前端输入如何影响动态建模、优化求解、权重决策与结果校验。

## 1. 当前系统流程

1. Step 1：输入工况参数，建立动态模型（DeepSeek 或本地规则库）。
2. Step 2：选择算法与运行档位，执行多目标优化（Web Worker）。
3. Step 3：查看 3D Pareto 前沿（E-C-Ra）。
4. Step 4：查看推荐解与 Top 5 候选解（TOPSIS 排序）。
5. Step 5：点击“校验优化结果”后，生成工程合理性与边界风险报告。

## 2. 参数输入与作用

### 2.1 齿轮基础参数

- `module`（模数）、`teeth`（齿数）、`faceWidth`（齿宽）决定切削体积、行程与机动时间。
- `accuracyGrade`（精度等级）直接影响粗糙度上限 `max_ra`。
- `hardness`（硬度）会影响切削力、寿命与推荐切削速度。

### 2.2 工件/刀具材料

系统会对输入文本做类别归一：

- 工件材料类别：碳钢、合金钢、渗碳钢、不锈钢、铸铁、其他。
- 刀具材料类别：高速钢、硬质合金、涂层硬质合金、陶瓷、其他。

材料组合影响：

- `tool_life_constant`、`tool_life_exponent`
- `specific_cutting_force`
- `max_cutting_speed`
- 进给上限（`f` 的可行上界）

### 2.3 机床与成本参数

- `maxPower` 写入约束 `constraints.max_power`，是硬约束核心。
- `machineRate`、`toolPrice`、`electricityRate`、`toolChangeTime`、`toolSharpeningCost`、`toolSharpeningLife` 进入成本目标函数。

## 3. 决策变量与边界

系统优化的四个决策变量：

- `d_a0`：滚刀直径
- `z_0`：滚刀头数
- `n`：主轴转速
- `f`：轴向进给量

其中 `z_0` 的规则非常关键：

- 当前下界固定为 `1`。
- 当前上界由精度等级控制：`accuracyGrade <= 8` 时上界为 `2`，否则上界为 `3`。
- 该边界在建模阶段生成，优化器仅在边界内搜索，并在计算时做整数化。

结论：

- 滚刀头数与刀具使用策略强相关，但当前实现主要由精度等级控制上界，不是前端单独手动输入项。
- 若需要手动控制 `z_0`，可在前端新增“头数范围”输入，再传入建模边界。

## 4. 可选算法配置（模块化）

算法不再硬编码，统一读取 `algorithm-config.js`。

- 算法配置文件：`algorithm-config.js`
- 算法实现目录：`algorithm/`
- 动态入口加载：`algorithm/index.ts`

新增算法推荐流程：

1. 在 `algorithm/` 新增算法文件（导出 `default` 或 `runAlgorithm`）。
2. 在 `algorithm-config.js` 新增一条算法配置（`id/entry/label/description` 等）。
3. 无需修改 `algorithm/index.ts`。

## 5. 多目标权重与 TOPSIS 评估

目标函数为：

- 能耗 `E`
- 成本 `C`
- 粗糙度 `Ra`

权重处理方式：

- 前端滑杆输入后先归一化（和为 1）。
- TOPSIS 内部先做向量归一化，再乘权重做综合评分。

这意味着：

- 三个目标量纲不同不会直接导致“大数值目标支配一切”。
- 调整权重只改变推荐解排序，不改变 Pareto 前沿本身。

## 6. 结果校验（Step 5）

点击“校验优化结果”后，系统会输出：

- 合理性结论：`合理 / 需关注 / 不合理`
- 约束一致性：功率、粗糙度、刀具寿命、速度区间、目标复算一致性
- 边界贴近检查：识别“贴边/触边”参数
- 异常信号与调参建议

校验来源：

- 优先：`AI + 规则`（有 DeepSeek Key）
- 回退：`规则引擎`

## 7. 建议使用方式

- 课堂演示优先使用标准牌号：`40Cr`、`20CrMnTi`、`W18Cr4V`。
- 若推荐解出现 `z_0` 触边，建议补充单头/双头方案做敏感性验证。
- 若功率或粗糙度利用率过高，优先回调 `n` 与 `f`，再复算校验。

## 8. 说明

- 本文档仅说明系统实现逻辑与参数含义，不替代现场试切规范。
- 生产应用建议结合机床实测、刀具厂商手册与质量检验数据二次确认。
